# ðŸ“– Configuration Management

The SiteWhere 2.0 architecture provides an externalized and highly-available configuration management subsystem based on Apache Zookeeper. Rather than storing configuration information locally, SiteWhere microservices connect to Zookeeper and query for the configuration that applies to them.

# Zookeeper Infrastructure

The SiteWhere Helm chart includes a template which is used to configure the Zookeeper components the system relies on. As part of the list of core infrastructure services required by SiteWhere, the Zookeeper pods must have been successfully deployed before any SiteWhere micrsoservices are allowed to start.
